# Agents Guide — apps/web

This is a Fumadocs (v16) + Next.js 16 app. Key differences from standard Next.js:

## Fumadocs Specifics

- Layouts use `LayoutProps<'/'>` type from fumadocs, not standard React props
- `RootProvider` from `fumadocs-ui/provider/next` wraps the app
- `HomeLayout` and `DocsLayout` are fumadocs layouts — don't recreate them
- MDX content lives in `content/docs/`, not in `src/app/docs/`
- `source.config.ts` at root configures MDX processing
- `.source/` directory is auto-generated by `fumadocs-mdx` — never edit it
- Search is handled by `src/app/api/search/route.ts`

## Do NOT

- Import from `.source/` directly — use `src/lib/source.ts` adapter
- Edit files in `.source/` — they are generated
- Use `next/font` Inter — we use Geist Sans and Geist Mono
- Create custom nav/sidebar — fumadocs handles this via `layout.shared.tsx`
- Use eslint — we use Biome

## Workspace Packages

This app imports from workspace packages:
- `@create-turbo-stack/core` — resolveFileTree(), wiring functions (browser-safe)
- `@create-turbo-stack/schema` — Zod schemas, types (browser-safe)
